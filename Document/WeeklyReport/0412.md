## 발표 피드백 및 주제 관련 회의

### 발표 피드백

#### 제안 아이디어에 대한 피드백

1. 질문 번역 to BERT 언어
2. 질문 재번역 to 지문 언어
3. 키워드 매칭 후 해당 단락 BERT 언어로 번역하여 QA 수행
4. (답 o) 해당 부분 질문 언어로 번역하여 제공 / (답 x) 전체 문서 번역 후 QA 재수행

위와 같이 번역이 빈번하게 수행되어, 오류, 노이즈 발생 가능성이 높아질 것이라 예상

또한 문제 상황을 크게 잡은 듯 (언어, 지문 제한 등을 통해 문제 규모를 줄여 테스트 해볼 것)

\+ QA에 대한 전반적인 설명(입력, 출력, 수행 과정 등), 주제에 대한 설명이 발표 첫 부분에 제시되어 이해를 용이하게 하면 좋을 것 같다.

### 회의 내용

#### 선행기술 조사, 발표 피드백을 통한 주제 구체화

금일(12일 화)까지 해당 주제에 대해 자료조사 진행

주제의 배경이 '다양한 언어를 지원하기 위해 BERT 모델을 여러개 두는 것은 비효율적이다.' 라는 전제였는데, 조사해보니 다언어를 지원하는 multilingual BERT가 존재했음

따라서 주제를 변경할지, 유지할지에 대해 회의하였음

> 결론 : 주제 유지

- 오히려 multilingual BERT와 비교하여 번역 API를 이용하여 더 넓은 스펙트럼의 언어를 지원할 수 있다라는 장점을 어필
- 또한 다언어를 지원하는 QA의 경우 영어 성능이 더 좋은 편향적인 결과가 나타나는데, 본 프로젝트에서 영어 BERT를 사용한다면 좋은 성능을 보일 수 있지 않을까 생각
- 오류, 노이즈 발생 가능성을 낮추는 것은 아직 생각되는 바 없음
- 언어 제한은 영어 / 한국어로 제한하고, 지문은 SQuAD / KorQuAD dataset으로 제한
- 평가를 영어 BERT, 한국어 BERT, multilingual BERT 와 비교하여 진행 
  + 가능한 질문-지문 쌍 : 영어-영어, 한국어-한국어, 영어-한국어, 한국어-영어
